# Video Lens

`Video Lens` 是一个本地运行的视频分析 Web 应用。

它解决的是一个很实际的问题：你可以直接选择一个视频，或者一次选择整个文件夹，让系统自动完成视频解析、调用视觉模型、返回结构化中文分析结果。用户不需要自己手动抽帧，也不需要自己写任何代码。

目前支持两条分析路径：

- `Ollama + qwen3-vl:8b`
  适合本地离线、隐私优先、希望把模型跑在自己电脑上的场景。
- `Gemini Video API`
  适合希望直接上传视频文件到云端视频接口的场景。

## 这个项目适合谁

- 想用 AI 分析本地视频，但没有太多代码基础的人
- 想批量分析一个文件夹视频的人
- 想对比本地模型和云端模型效果的人
- 想把“视频理解”做成一个可操作的小工具，而不是命令行实验的人

## 功能

- 单视频分析
- 整文件夹批量分析
- 实时任务状态和日志
- 每个视频独立结果查看
- 本地 `Ollama` 路线
- 云端 `Gemini` 路线
- 中文结果输出

## 项目截图式理解

你可以把它理解成一个“审片工作台”：

1. 左侧设置任务参数
2. 右侧看当前任务状态
3. 下方左边看批量队列
4. 下方右边看当前视频的详细分析结果

## 运行前你需要准备什么

### 1. 安装 Python

本项目后端使用 Python，自带标准库启动服务，不需要安装额外第三方 Python 包。

推荐版本：

- `Python 3.11+`

检查方法：

```bash
python3 --version
```

如果没有安装：

- macOS：去 [Python 官网](https://www.python.org/downloads/) 安装，或者使用 `brew install python`
- Windows：去 [Python 官网](https://www.python.org/downloads/windows/) 安装，安装时勾选 `Add Python to PATH`

### 2. 安装 ffmpeg 和 ffprobe

本项目需要 `ffmpeg` 和 `ffprobe` 来读取视频信息、抽取关键帧。

检查方法：

```bash
ffmpeg -version
ffprobe -version
```

如果没有安装：

- macOS（Homebrew）：

```bash
brew install ffmpeg
```

- Windows（winget）：

```powershell
winget install Gyan.FFmpeg
```

- Linux（Ubuntu / Debian）：

```bash
sudo apt update
sudo apt install ffmpeg
```

### 3. 如果你要用本地模型，请安装 Ollama

下载地址：

- [Ollama 官网](https://ollama.com/download)

安装完成后，启动 `Ollama`。

检查是否成功运行：

```bash
ollama list
```

或者：

```bash
curl http://127.0.0.1:11434/api/tags
```

如果你能看到返回结果，说明本地服务已经在运行。

### 4. 拉取本项目默认使用的模型

本项目默认本地模型是：

- `qwen3-vl:8b`

拉取命令：

```bash
ollama pull qwen3-vl:8b
```

拉完以后再执行：

```bash
ollama list
```

确认列表里出现 `qwen3-vl:8b`。

### 5. 如果你要用 Gemini 路线，需要 API Key

如果你只用本地 `Ollama`，这一步可以跳过。

如果你要用 `Gemini` 路线，需要准备一个 Google AI Studio / Gemini API Key。

你后续只需要在网页里填入：

- `Gemini API Key`
- `Gemini 模型名`

默认模型是：

- `gemini-2.5-pro`

## 第一步一步跑通项目

### 1. 克隆仓库

```bash
git clone <你的仓库地址>
cd <仓库目录>
```

### 2. 启动 Ollama

如果你走本地模型路线，先确认 `Ollama` 已经启动，并且 `qwen3-vl:8b` 已经拉取完成。

### 3. 启动项目

```bash
python3 app.py
```

看到类似输出：

```text
Video Lens running at http://127.0.0.1:8765
```

说明启动成功。

### 4. 打开网页

在浏览器访问：

```text
http://127.0.0.1:8765
```

### 5. 开始使用

你会看到一个网页工作台。

#### 如果你想分析一个视频

1. 选择 `单个视频`
2. 选择 `Ollama` 或 `Gemini`
3. 选择一个视频文件
4. 可选填写你的附加要求
5. 点击 `开始分析`

#### 如果你想批量分析整个文件夹

1. 选择 `整个文件夹`
2. 选择 `Ollama` 或 `Gemini`
3. 选择一个包含多个视频的文件夹
4. 可选填写你的附加要求
5. 点击 `开始批量分析`

系统会自动：

- 找出文件夹中的视频文件
- 按顺序处理
- 为每个视频生成独立结果
- 在队列中显示进度和状态

## 如何写提示词

你可以在“附加要求”里写你想让模型重点关注的东西。

例如：

```text
请按时间顺序总结视频内容，列出出现的人物、场景、动作，并指出你不确定的地方。
```

或者：

```text
请重点分析视频中的产品展示、人物动作和镜头变化。
```

或者：

```text
请帮我判断视频是否存在明显的广告感、摆拍感或场景不一致问题。
```

## Ollama 路线到底是怎么工作的

很多人会问：

“我不是直接选了视频吗？为什么还能用本地视觉模型分析？”

这里要区分两层：

- 用户使用层：你直接选视频文件
- 应用内部实现层：程序自动读取视频、抽取关键帧、把关键帧发给视觉模型

也就是说，用户不需要手动抽帧，但应用内部会自动完成这一步。

这就是本项目存在的价值：把复杂的视频预处理流程包装成一个普通用户可以直接操作的界面。

## Gemini 路线怎么理解

Gemini 路线下，用户仍然是“直接上传视频文件”，不需要自己先转图片。

因此从用户视角看，它就是一个直接接受视频文件的视频理解接口。

## 常见问题

### 1. 页面显示 Ollama 未就绪怎么办？

先检查：

```bash
ollama list
```

如果命令没有返回模型列表，说明 Ollama 还没有正常运行。

### 2. 提示找不到 `qwen3-vl:8b` 怎么办？

执行：

```bash
ollama pull qwen3-vl:8b
```

### 3. 提示找不到 ffmpeg / ffprobe 怎么办？

说明你的系统没有安装视频工具，按照前面的 `ffmpeg` 安装步骤安装即可。

### 4. 为什么批量分析比较慢？

因为当前是“逐个视频顺序处理”，这样更稳定，也更容易看到每个视频的独立状态和错误。

### 5. 为什么本地模型分析速度不快？

本地视觉模型速度主要受这些因素影响：

- 电脑性能
- 模型大小
- 视频长度
- 抽帧数量
- Ollama 运行状态
